// 01 — Foundations

What Is a SOC?

A Security Operations Center (SOC) is a centralized team responsible for continuously monitoring, detecting, analyzing, and responding to cybersecurity events across an organization's infrastructure. SOC analysts are the people watching the screens — correlating alerts, hunting for threats, and escalating incidents before they become breaches.

Core competency: A good SOC analyst isn't just reactive — they understand attacker techniques, know what normal looks like, and can distinguish real threats from noise. That judgment comes from study and hands-on practice.

Foundational Reading

These are the industry-standard references that define how professional SOCs are built and run. Start with MITRE Part 0, then work through the others as your schedule allows.

  • 📘
    MITRE — 11 Strategies for a World-Class SOC Start with Part 0: Fundamentals. This is the definitive guide to SOC strategy, written by practitioners. Covers mission, people, process, and technology across 11 evidence-backed strategies. Free PDF also available on GitHub.
  • 📗
    FIRST — Building a SOC: Start Small A concise, practical guide from the Forum of Incident Response and Security Teams. Ideal for understanding how to bootstrap a SOC from limited resources — directly applicable to our university context.
  • 📙
    NCSC (UK) — Building a Security Operations Centre The UK National Cyber Security Centre's SOC guidance. Clear, actionable, and vendor-neutral. Covers scope, detection capability, staffing models, and tooling choices.
  • 📕
    Gartner — SOC Model Guide 2023 Industry analyst perspective on SOC maturity models, staffing tiers (L1/L2/L3), and technology stack evolution. Useful for understanding how enterprise SOCs are structured.
  • 📊
    Splunk — State of Security 2023 Annual survey of security practitioners on challenges, tooling, burnout, and emerging threats. Gives you a realistic picture of what the security field looks like today.
  • 🔐
    Microsoft — Secure Your Business with Microsoft 365 Best practices for M365 security configuration. Relevant because many organizations (including universities) depend on M365 — understanding its security surface is useful context.

// 02 — Access

Getting Into ELK via the Cyber Range

All SOC work happens inside the campus cyber range. Follow these steps to get set up before your first shift. Contact your instructor or SOC lead if you run into access issues.

Enroll in the Cyber Range

Request access to the university cyber range through your course enrollment or by contacting the SOC program coordinator. You'll receive credentials and VPN configuration instructions by email.

Connect via VPN

The cyber range environment is not accessible from the public internet. Install and configure the provided VPN client, then connect to the range network before attempting to reach any internal resources.

# Connect to the range (example — use your actual endpoint)
$ openvpn --config cyberrange.ovpn

Open Kibana in Your Browser

Once on the VPN, navigate to the Kibana URL provided by your SOC coordinator (internal IP, port 5601 by default). Log in with your analyst credentials.

# Kibana default address (confirm with your coordinator)
$ http://<range-ip>:5601

Verify Log Ingestion

In Kibana, navigate to Discover and confirm you can see recent log events from the Palo Alto firewall. If the index shows no data, check with your SOC lead — ingestion may need to be verified.

Review the Shift Dashboard

Load the SOC shift dashboard in Kibana. This is your primary working view — it surfaces high-priority alerts, threat events, and traffic anomalies for the current monitoring window.

// 03 — ELK

Basic ELK Navigation

ELK (Elasticsearch, Logstash, Kibana) is our primary analyst interface. You'll spend most of your shift time inside Kibana. The links below cover the core workflows you'll use.

Discover

Raw log search. Use KQL to filter events by IP, action, threat category, or time range. Your main triage surface.

→ Elastic Discover Docs

Dashboards

Pre-built and custom visualizations of your log data. Use the shift dashboard to get an at-a-glance view of threat volume.

→ Elastic Dashboard Docs

KQL Reference

Filter on fields like event.action and source.ip. Learn the basics before your first shift.

→ KQL Query Language Docs

Alerts & Rules

SOC leads manage detection rules; analysts review and triage the results in Kibana's alerting layer.

→ Alerting Docs

Tip: When you find an interesting event in Discover, use the surrounding documents feature to see what was happening before and after — context is everything in triage.
