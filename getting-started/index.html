// 01 — Foundations

What Is a SOC?

A Security Operations Center (SOC) is a centralized team responsible for continuously monitoring, detecting, analyzing, and responding to cybersecurity events across an organization's infrastructure. SOC analysts are the people watching the screens — correlating alerts, hunting for threats, and escalating incidents before they become breaches.

Core competency: A good SOC analyst isn't just reactive — they understand attacker techniques, know what normal looks like, and can distinguish real threats from noise. That judgment comes from study and hands-on practice.

// 02 — Access

Getting Into ELK via the Cyber Range

All SOC work happens inside the campus cyber range. Follow these steps to get set up before your first shift.

Enroll in the Cyber Range

Request access through your course enrollment or by contacting the SOC program coordinator. You'll receive credentials and VPN configuration instructions by email.

Connect via VPN

The cyber range is not accessible from the public internet. Install and configure the provided VPN client before attempting to reach any internal resources.

# Connect to the range (use your actual endpoint)
$ openvpn --config cyberrange.ovpn

Open Kibana in Your Browser

Once on the VPN, navigate to the Kibana URL provided by your SOC coordinator (internal IP, port 5601 by default).

# Kibana default address (confirm with your coordinator)
$ http://<range-ip>:5601

Verify Log Ingestion

In Kibana, navigate to Discover and confirm you can see recent log events from the Palo Alto firewall. If the index shows no data, check with your SOC lead.

Review the Shift Dashboard

Load the SOC shift dashboard in Kibana — your primary working view for high-priority alerts, threat events, and traffic anomalies.

→ Elastic Discover Docs

Dashboards

Pre-built and custom visualizations. Use the shift dashboard to get an at-a-glance view of threat volume and blocked traffic.

→ Elastic Dashboard Docs

KQL Reference

Filter on fields like event.action and source.ip. Learn the basics before your first shift.

→ KQL Docs

Alerts & Rules

SOC leads manage detection rules; analysts review and triage results in Kibana's alerting layer.

→ Alerting Docs

Tip: When you find an interesting event in Discover, use the surrounding documents feature to see what happened before and after — context is everything in triage.
