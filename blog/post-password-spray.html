Threat Report May 2026 — Cybercave SOC Team

Password Spray Activity Detected on Campus VPN

Summary

During routine monitoring, SOC analysts identified a low-and-slow password spray attack targeting the campus VPN authentication endpoint. Unlike brute-force attacks that hammer a single account with many passwords, password spraying tries one or two common passwords across many usernames — designed to stay under account lockout thresholds and avoid triggering alerts.

What We Observed

Over a 72-hour window, analysts noted repeated authentication attempts against the VPN gateway from a small set of external IP addresses. Each source IP attempted authentication against dozens of different usernames using a handful of commonly-used passwords (e.g. seasonal passwords, "Password123" variants), with several hours between attempts per account — a pattern designed to evade lockout policies and rate-based detections.

Indicators of Compromise

# Example log pattern observed in Kibana
2026-05-12 03:14:02 UTC src=185.220.101.42 dst=vpn-gateway action=auth_fail user=jsmith24
2026-05-12 03:14:09 UTC src=185.220.101.42 dst=vpn-gateway action=auth_fail user=mwilliams7
2026-05-12 03:14:17 UTC src=185.220.101.42 dst=vpn-gateway action=auth_fail user=rgarcia3
# Pattern: one attempt per username, ~7-second interval, no repeats

Response Actions Taken

Indicator enrichment

Source IPs were checked against AbuseIPDB and OTX — all were flagged with prior reports of brute-force and credential-stuffing activity from VPN/RDP scanning campaigns.

Perimeter block

Identified source IPs were added to the firewall block list. Recommendation submitted to IT security for review and longer-term blocking.

Account review

No successful authentications were observed from the flagged source IPs during the activity window. No accounts required forced password resets.

Detection rule deployed

A new Kibana detection rule was created to flag distributed authentication failure patterns against the VPN gateway — multiple distinct usernames from a single source IP within a short window.

Reminder for students and staff: This is exactly why password reuse and weak passwords matter. If your VPN password is also used elsewhere — or follows a predictable pattern — now is a good time to update it. See the Resources page for password management guidance.
