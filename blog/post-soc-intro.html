SOC Update April 2026 — ElDiyablo

Introducing the Cybercave SOC Documentation Site

For a while now, Cybercave SOC knowledge has lived scattered across Canvas pages, shared drives, and Discord threads. That works fine when you're the one who wrote it — it works a lot less well for the next analyst trying to find it. So we built something better: a centralized, public documentation site for the SOC, hosted right alongside the code that runs it.

What's on the Site

🚀

Getting Started

Everything a new analyst needs before their first shift — what a SOC is, how to access ELK through the cyber range, and how to navigate Kibana and OTX.

🛠

Tools

The full analyst toolkit organized by workflow phase — SIEM, threat intel enrichment, detection frameworks, and training platforms.

📋

Ticketing

How to write tickets that actually help the next person — structure, evidence, enrichment, and recommended actions.

📚

Resources

Security guidance for the whole student body — phishing red flags, password management, and everyday security habits.

Why It's on GitHub Pages

Hosting this on GitHub keeps it close to the code and tools we already use, makes it easy for any analyst to contribute via pull request, and gives us version history for free — every change to our processes is tracked. It also means the site is publicly accessible, which fits our values around transparency: there's nothing here that needs to be hidden, and sharing it openly helps other student SOC programs get off the ground faster.

What's Next

This is a living site. Coming up: automated deployment via GitHub Actions, a security audit and SECURITY.md policy, and a full user guide so any analyst — not just the person who built it — can update content with confidence. If you're a SOC analyst and want to contribute a guide, tool writeup, or advisory, see the Blog template and open a pull request.

Have feedback? This site exists for analysts and students. If something's missing, unclear, or broken, let the SOC lead know — or better yet, fix it and submit a PR.
